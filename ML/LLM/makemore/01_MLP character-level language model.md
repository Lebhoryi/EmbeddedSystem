- 验证集：用于模型的调优过程，在训练过程中，模型并不会用验证集来更新参数，而是用来评估模型在未见过的数据上的表现。
  - 用于超参数调整，例如学习率、网络层数、正则化系数
  - 用于早停，防止过拟合：如果模型在训练集上的表现越来越好，但是在验证集上表现变差，说明模型开始记忆训练数据而不是泛化规律
- 测试集：在模型的训练和调优都完成后，才使用测试集来评估模型的泛化能力（在未见过的数据上的表现）
  - 用于模型的最终评估，测试集在整个开发过程中都不能被用来训练或调优模型

## 构建神经网络

### 模型结构

我们准备构建一个前馈神经网络，具体为带有嵌入层的单隐藏层 MLP。

- 嵌入层：C (27x10) 将输入索引映射到 10 维向量，不涉及激活函数，常视为预处理层。

- 隐藏层：W1 (30x200) + b1 (200) + tanh 激活函数，这是唯一的隐藏层（非线性变换）。

- 输出层：W2 (200x27) + b2 (27)，无激活（直接输出 logits），后接 `softmax` 或 `cross-entropy`

上下文：输入上下文长度为 3（block_size=3），嵌入后展平为 30 维（3x10），然后进入隐藏层。这是一个浅层网络，适合简单任务如名字生成。

# 前馈神经网络

前馈神经网络（Feedforward Neural Network, FFNN）是神经网络的最基本类型之一，常用于分类、回归等任务。

### 1. **定义**

- 前馈神经网络是一种信息单向流动的神经网络模型，其中数据从输入层经过隐藏层直接流动到输出层，没有反馈循环（即没有从输出返回到输入的路径）。
- 它也被称为“多层感知机”（Multi-Layer Perceptron, MLP）当它有多个隐藏层时。核心思想是模拟大脑的神经元连接，通过层级结构学习输入到输出的映射。

### 2. **结构**
- **输入层 (Input Layer)**：接收原始数据（如向量、图像像素）。每个节点代表一个输入特征。
- **隐藏层 (Hidden Layers)**：一或多个中间层，每个层包含多个神经元（节点）。每个神经元通过权重矩阵和偏置进行线性变换，然后应用激活函数（如 sigmoid、ReLU 或 tanh）引入非线性。
- **输出层 (Output Layer)**：产生最终结果（如分类概率、回归值）。通常使用 softmax（分类）或线性激活（回归）。
- **连接方式**：全连接（Fully Connected），即每层的所有神经元都与下一层的所有神经元相连。信息流动是单向的：输入 → 隐藏 → 输出，没有循环。

示例结构（简单 2 层 FFNN）：
```
输入层 (e.g., 3 维) → 隐藏层 (e.g., 200 神经元, tanh) → 输出层 (e.g., 27 维)
```

### 3. **工作原理**
- **前向传播 (Forward Propagation)**：数据从输入层逐层向前传递。
  - 对于每个层：输出 = 激活函数(权重矩阵 × 输入 + 偏置)。
  - 示例数学表示：隐藏层 h = tanh(W1 * x + b1)，输出 o = W2 * h + b2。
- **训练过程**：使用反向传播 (Backpropagation) 和梯度下降优化参数（权重和偏置），最小化损失函数（如交叉熵）。
- **无循环**：不像循环神经网络 (RNN)，它不处理时间序列或反馈；每一步计算都是独立的。

### 4. **特点和优缺点**
- **优点**：
  - 简单易实现，计算效率高。
  - 适合静态输入（如图像分类、数值预测）。
  - 通用性强，能逼近任何连续函数（通用逼近定理）。
- **缺点**：
  - 无法捕捉序列或时间依赖（e.g., 语言中的上下文），因为没有记忆机制。
  - 对于复杂任务，可能需要很深的层，导致梯度消失/爆炸问题。
- **与其它网络的区别**：
  - **对比 RNN**：RNN 有循环，能处理序列数据（如文本），但可能有梯度问题。
  - **对比 CNN**：CNN 专为网格数据（如图像）设计，使用卷积层共享权重。
  - **对比 Transformer**：Transformer 使用自注意力机制，无循环但能并行处理序列，比 FFNN 更先进。

#  cross_entropy 与 softmax

`cross_entropy`（交叉熵）和 `softmax` 是深度学习中两个相关但**完全不同**的概念，它们分别扮演不同的角色，通常一起使用，但功能截然不同。

------

## 一、Softmax 是什么？

**Softmax** 是一个**函数**，它的作用是将一组**原始分数（logits）转换为概率分布**。

### 定义：

给定一组 logits（通常是神经网络最后一层的原始输出，未归一化的数值），Softmax 函数会将每一个 logit 转换为一个 **0 到 1 之间的值**，并且所有这些值的**和为 1**，形成一个概率分布。

对于一个向量 $\mathbf{z} = [z_1, z_2, ..., z_n]$，第 $i$ 个类别的 Softmax 输出为：
$$
P_i = \frac{e^{z_i}}{\sum_{j=1}^{n} e^{z_j}}
$$

### 特点：

- 将任意实数值的 logits，转换为概率。
- 适用于**多分类问题**，表示每个类别的“可能性”。
- 强调分数高的值对应的概率更大，且差异被放大。

🔍 **Softmax 本身不做损失计算，它只是将网络的原始输出变成概率。**

------

## 二、Cross Entropy（交叉熵）是什么？

**Cross Entropy** 是一种**损失函数（Loss Function）**，用于衡量**预测的概率分布与真实分布之间的差距**，常用于分类任务，尤其是**分类的监督学习任务**。

### 直观理解：

假设你有一个真实的标签（比如图片是一只猫，对应类别 2），你的模型输出了一个概率分布（比如 [0.1, 0.2, 0.7]，表示它认为分别是类别 1、2、3 的概率）。交叉熵就是用来衡量这个预测分布与真实分布之间有多“不匹配”。

### 公式（针对单个样本，真实标签为 one-hot）：

如果真实类别是第 $k$ 类（使用 one-hot 编码，比如 [0, 0, 1] 表示第3类），模型预测该类别的概率为 $p_k$，则交叉熵损失为：
$$
H(p, q) = - \log(p_k)
$$
更一般地，如果真实分布是 $y$（one-hot），预测分布是 $\hat{y}$，则：
$$
H(y, \hat{y}) = - \sum_{i} y_i \log(\hat{y}_i)
$$

### 特点：

- 衡量的是两个概率分布之间的差异。
- 在分类任务中，通常真实标签是 one-hot 形式，预测是 softmax 后的概率。
- **交叉熵越小，表示预测越接近真实。**

------

## 三、Softmax 和 Cross Entropy 的关系

在实际应用中，尤其是做**多分类任务时**，我们经常将 **Softmax 和 Cross Entropy 结合在一起使用**：

### 常见做法：

1. **神经网络最后一层输出 logits（未归一化的分数）**
2. 对这些 logits 应用 **Softmax**，得到每个类别的概率 $\hat{y}$
3. 将这个概率分布 $\hat{y}$ 与真实标签（通常为 one-hot）计算 **Cross Entropy Loss**

🔧 很多深度学习框架（如 PyTorch、TensorFlow）为了**数值稳定性与计算效率**，提供了一个 **组合函数**，比如：

- `torch.nn.CrossEntropyLoss` （**内部已经包含了 softmax 操作！**）
- `tf.keras.losses.CategoricalCrossentropy` （有时也需要先经过 softmax）

⚠️ **注意：**

- 如果你使用类似 PyTorch 的 `CrossEntropyLoss`，**不需要手动加 softmax**，因为它在内部已经实现了 softmax + cross entropy，并做了数值稳定的优化。
- 如果你**先手动做了 softmax，再计算交叉熵**，在一些框架中可能效率不高或者不够数值稳定。

------

## 四、总结对比表

| 项目                 | Softmax                                    | Cross Entropy                              |
| -------------------- | ------------------------------------------ | ------------------------------------------ |
| **是什么**           | 一个函数，将 logits 转为概率分布           | 一种损失函数，衡量预测分布与真实分布的差异 |
| **输入**             | 原始网络的输出（logits，任意实数）         | 预测的概率分布 + 真实标签（通常 one-hot）  |
| **输出**             | 每个类别的概率，范围 [0, 1]，且总和为 1    | 一个标量值，表示预测与真实之间的“损失”     |
| **用途**             | 将模型输出转化为概率，用于分类任务中的预测 | 衡量分类预测的好坏，用于训练时优化模型     |
| **是否用于训练**     | 一般不直接用于损失计算，而是作为中间步骤   | ✅ 是，用于反向传播和优化                   |
| **是否需要真实标签** | ❌ 不需要                                   | ✅ 需要，通常是 one-hot 或类别索引          |
| **常见搭配**         | 与 Cross Entropy 搭配使用                  | 通常与 Softmax 的输出搭配使用              |

# PCA 与 t-SNE

**PCA（主成分分析，Principal Component Analysis）** 和 **t-SNE（t-Distributed Stochastic Neighbor Embedding）** 都是常用于**降维（Dimensionality Reduction）**的算法，尤其是在数据可视化、特征提取等任务中非常常见。但它们在**原理、目的、应用场景和输出效果**上有很大区别。

------

## 一、一句话直观对比

| 方法         | PCA                                    | t-SNE                                         |
| ------------ | -------------------------------------- | --------------------------------------------- |
| **类型**     | 线性降维方法                           | 非线性降维方法                                |
| **主要目的** | 最大化保留数据的**全局方差（结构）**   | 最大化保留数据的**局部相似性（邻居关系）**    |
| **是否线性** | ✅ 线性                                 | ❌ 非线性                                      |
| **适用场景** | 数据压缩、去除冗余、初步探索性分析     | 数据可视化（特别是高维到2D/3D）、观察聚类结构 |
| **输出特点** | 保持数据的全局结构，但可能忽略局部聚类 | 能很好展示局部聚类，但全局结构可能混乱        |

------

## 二、详细对比

### 1. PCA（主成分分析）

#### ✅ 原理：

PCA 是一种**线性降维方法**，它的目标是找到一个**新的坐标系（主成分）**，使得数据在新坐标系下的**投影方差最大化**，也就是尽可能保留数据中的**最大变化方向**。

- 它通过计算数据的**协方差矩阵**，然后对该矩阵进行**特征值分解**，选取**前几个最大的特征值对应的特征向量**作为主成分方向。
- 选择前 k 个主成分，就可以将原始高维数据投影到这 k 个方向上，达到降维目的。

#### 🧠 特点：

- **线性变换**：新特征是原特征的线性组合。
- **保留全局结构**：关注的是整体数据的方差/分布，而不是局部点与点之间的关系。
- **计算高效**，适合处理大规模数据。
- 结果具有**可解释性**（每个主成分对应原始变量的线性组合）。

#### 📌 应用场景：

- 数据压缩
- 去除冗余特征
- 数据可视化（例如降到2D/3D，但不如 t-SNE 直观）
- 特征提取与降噪

#### 🚩 局限：

- 对于**非线性结构**的数据（比如流形结构），PCA 很难捕捉到数据的真实分布。
- 不能很好地展示**局部聚类或相似性**

------

### 2. t-SNE（t-分布随机邻域嵌入）

#### ✅ 原理：

t-SNE 是一种**非线性降维方法**，特别擅长将高维数据可视化到**2D 或 3D 空间**，目标是**保留数据点之间的局部相似性（即邻居关系）**。

- 它的核心思想是：**在高维空间中相近的点，在低维空间中也尽量相近；而高维空间中远离的点，不一定要求在低维中也远离**。
- 具体来说：
  - **Step 1**：计算高维空间中每对数据点之间的相似性（通常用高斯分布计算概率）。
  - **Step 2**：在低维空间（比如2D）中也构建类似的点，并定义相似性（通常用 t-分布，即 heavy-tailed 分布）。
  - **Step 3**：通过优化一个损失函数（KL 散度），使得高维和低维中的这些“邻居概率分布”尽可能接近。

#### 🧠 特点：

- **非线性方法**：能捕捉复杂的流形结构、聚类等。
- **擅长展示局部结构**：比如聚类、分组等，可视化效果非常好。
- **对全局结构不敏感**：不同簇之间的相对位置可能没有意义，甚至可能混乱。
- **计算复杂度高**，尤其是数据量大时训练慢。
- **结果不可直接解释**，更多用于观察模式，而非特征提取。

#### 📌 应用场景：

- **高维数据可视化（如图像特征、词向量、生物数据等）**
- 观察数据的**聚类效果**
- 探索数据的潜在结构

#### 🚩 局限：

- 对**全局结构**保留较差（比如类别之间的相对关系可能丢失）
- 结果可能不稳定（对超参数如困惑度 perplexity 敏感）
- **不适合直接用于降维后的特征再利用**（比如再输入到机器学习模型中）

------

## 三、对比总结表

| 特性                     | PCA                                | t-SNE                                    |
| ------------------------ | ---------------------------------- | ---------------------------------------- |
| **类型**                 | 线性降维                           | 非线性降维                               |
| **目标**                 | 最大化保留数据的总体方差           | 最大化保留局部点之间的相似性（邻居关系） |
| **计算速度**             | 快，适合大数据                     | 慢，尤其大数据时训练成本高               |
| **可解释性**             | 可解释（主成分是原特征的线性组合） | 不可解释，结果更偏向可视化               |
| **保留结构**             | 全局结构（整体分布）               | 局部结构（邻近点关系）                   |
| **全局结构**             | 保留较好                           | 通常丢失或混乱                           |
| **适用场景**             | 特征提取、数据压缩、初步分析       | 数据可视化、聚类观察                     |
| **是否常用作可视化工具** | 可用，但不如 t-SNE 直观            | ✅ 非常适合高维数据可视化                 |
| **是否适合做特征再利用** | ✅ 可以，降维后的特征仍可用         | ❌ 一般不用于特征工程                     |

------

## 四、什么时候用 PCA？什么时候用 t-SNE？

### ✅ **使用 PCA 的情况：**

- 你想要**快速降维**，比如从 1000 维降到 50 维，用于减少计算量或去除噪声。
- 你关心的是数据的**整体方差和主要变化方向**。
- 你想要做**特征提取或数据压缩**，并且降维后的特征还要**输入到其他模型中继续使用**。
- 数据量较大，需要**快速计算**。

### ✅ **使用 t-SNE 的情况：**

- 你想要**可视化高维数据（比如 50~1000 维）到 2D/3D**，观察数据的聚类、分布情况。
- 你关心的是数据点之间的**局部相似性、是否形成簇、分组是否合理**。
- 你做探索性数据分析（EDA），想直观地理解数据的结构。
- 数据量不是特别大（t-SNE 对大规模数据计算较慢，但也有优化版如 **Barnes-Hut t-SNE** 或 **UMAP**）。

------

## 五、额外提示：PCA + t-SNE 组合使用

在实际应用中，尤其是数据可视化时，一个常见的流程是：

> **先用 PCA 降维到一个中等维度（比如 50 维或 30 维），去除噪声和冗余，然后再用 t-SNE 降到 2D/3D 做可视化。**

🔧 原因：

- t-SNE 对输入数据的尺度、冗余比较敏感，先使用 PCA 可以提高效率和效果。
- PCA 能快速去除一些无关维度，让 t-SNE 更聚焦于有意义的特征。

------

## 六、一句话总结

> **PCA 是一种快速、线性的降维方法，适合保留数据的全局结构，常用于特征提取和数据压缩；而 t-SNE 是一种非线性、复杂但强大的可视化工具，擅长展示数据的局部聚类与相似性，常用于高维数据探索和可视化。**
